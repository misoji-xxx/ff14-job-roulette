# FF14 ジョブルーレット 仕様書

## 概要

FF14のパーティメンバーにランダムでジョブを割り当てるWebアプリケーション。
GitHub Pagesで公開予定。

---

## モード一覧

| モード | 説明 | 状態 |
|--------|------|------|
| シンプルモード | すぐにルーレットが実行できる | 実装済み |
| 高機能モード | 詳細な設定が可能 | 実装済み |

---

## シンプルモード

### 機能一覧

| 機能 | 説明 |
|------|------|
| パーティ人数選択 | 4人または8人を選択 |
| プレイヤー名入力 | 各メンバーの名前を入力（空欄時はデフォルト名） |
| ルーレット実行 | ボタン押下でランダムにジョブを割り当て |
| 結果表示 | ジョブアイコン・名前・ロールを表示 |

### パーティ構成

| 人数 | タンク | ヒーラー | DPS |
|------|--------|----------|-----|
| 4人  | 1      | 1        | 2   |
| 8人  | 2      | 2        | 4   |

### UI仕様

- プレイヤー入力欄にはフリーアイコン（グレー）を表示
- 結果表示はロールごとに色分け（タンク: 青、ヒーラー: 緑、DPS: 赤）

### 仕様上の注意事項

- どのプレイヤーがどのロールになるかはランダムに決定される（パーティ構成は維持）
- ジョブの重複割り当ては許可される（同じジョブが複数人に割り当てられる場合がある）
- パーティ人数を変更すると入力済みのプレイヤー名はリセットされる

## 高機能モード

### モード切り替え

- 画面上部に「シンプル / 高機能」のトグルボタンを設置
- クリックでモードを切り替え

### パーティ構成

#### 人数選択

- 4人 / 8人から選択（シンプルモードと同様）

#### 枠ごとのロール設定

各プレイヤー枠に対してロールを個別に設定できる。

| 人数 | 選択可能なロール |
|------|------------------|
| 4人 | タンク / ヒーラー / DPS / フリー |
| 8人 | タンク / ヒーラー / ピュアヒーラー / バリアヒーラー / DPS / メレー / レンジ / キャスター / フリー |

- **フリー**：ルーレット時にランダムでロールが決定される
- **ヒーラー**（8人）：ピュア/バリアを問わずヒーラーを割り当て
- **ピュアヒーラー**：白魔道士、占星術師のみ
- **バリアヒーラー**：学者、賢者のみ
- **DPS**（8人）：メレー/レンジ/キャスターを問わずDPSを割り当て
- ロール選択ボタンをクリックするとモーダルが開く
- モーダル内でカテゴリ（タンク/ヒーラー/DPS/その他）ごとにロールを選択
- 選択したロールに応じてアイコンと背景色が変化する

#### 初期値

人数選択時・モード切り替え時の初期値は全員「フリー」。

### 詳細設定

#### ジョブ重複禁止

| 状態 | 動作 |
|------|------|
| ON | 同じジョブが複数人に割り当てられない |
| OFF | 同じジョブが複数人に割り当てられる可能性がある（デフォルト） |

#### 標準構成維持

汎用枠（フリー/ヒーラー/DPS）のロール決定時に、標準的なパーティ構成を維持するオプション。

| 人数 | 標準構成 |
|------|----------|
| 4人 | 1T、1H、2DPS |
| 8人 | 2T、1ピュアH、1バリアH、2メレー、1レンジ、1キャス |

| 状態 | 動作 |
|------|------|
| ON | 汎用枠のロールは標準構成を満たすように決定される（デフォルト） |
| OFF | 汎用枠は完全ランダム（構成が偏る可能性あり） |

##### 汎用枠の解決ロジック（8人・標準構成維持ON）

汎用枠は以下の優先順位でサブロールに解決される：

1. **汎用ヒーラー枠** → 不足しているヒーラーサブロール（ピュア/バリア）に割り当て
2. **汎用DPS枠** → 不足しているDPSサブロール（メレー/レンジ/キャス）に割り当て
3. **フリー枠** → 残りの不足分（タンク含む）に割り当て

###### 例：タンクx1、ピュアx1、ヒーラーx1、メレーx2、レンジx1、DPSx1、フリーx1

| 枠 | 不足 | 解決 |
|----|------|------|
| ヒーラー（汎用） | バリア1 | → バリアヒーラー |
| DPS（汎用） | キャス1 | → キャスター |
| フリー | タンク1 | → タンク |

##### 選択可否の条件

- 固定枠が標準構成を超過している場合は非活性（選択不可）
  - 例：8人構成でタンク固定が3人以上 → 標準構成を満たせないため非活性
  - 非活性時は「（固定枠が超過）」と表示

### プレイヤー個別設定

#### ジョブ除外

- 各プレイヤーに対して、割り当てから除外するジョブを設定できる
- 除外ボタン（🚫アイコン）をクリックするとモーダルが開く
- モーダル内で除外したいジョブをクリックして選択（複数選択可）
- 除外設定があるプレイヤーは、除外ボタンに件数バッジが表示される

### 結果表示

#### ロール表示ルール

| モード | ロール設定 | 標準構成維持 | 表示 |
|--------|-----------|-------------|------|
| シンプル | - | - | タンク / ヒーラー / DPS |
| 高機能 | 明示的に設定 | - | 設定したロール（メレー等） |
| 高機能 | フリー | ON | サブロール（メレー / ピュアヒーラー等） |
| 高機能 | フリー | OFF | タンク / ヒーラー / DPS |

- 明示的にロールを設定した場合は、標準構成維持の設定に関わらず設定したロールが表示される
- フリー枠から割り当てられた場合のみ、標準構成維持の設定に応じて表示が変わる

### エラー処理

以下の場合はエラーメッセージを表示：

- ジョブ除外や重複禁止の設定により、割り当て可能なジョブがなくなった場合
  - 例：「○○ に割り当て可能なジョブがありません。除外設定やジョブ重複禁止設定を見直してください。」

## 結果コピー機能

### 概要

ルーレット結果をテキスト形式でクリップボードにコピーできる機能。
シンプルモード・高機能モードの両方で利用可能。

### UI仕様

| 要素 | 説明 |
|------|------|
| コピーボタン | 結果一覧の下部に「結果をコピー」ボタンを表示 |
| アイコン | 📋（クリップボード）アイコン付き |
| フィードバック | コピー成功時はボタンが緑色に変化し「コピーしました」と表示（2秒後に戻る） |

### コピー形式

```
プレイヤー1: 🔫ガンブレイカー
プレイヤー2: 🤍白魔道士
プレイヤー3: ⚔️侍
プレイヤー4: 🔥黒魔道士
```

- 各行は「プレイヤー名: 絵文字ジョブ名」の形式
- ロール表記は含まない（シンプルに保つ）
- 各ジョブに対応する絵文字が先頭に付く

## 再抽選機能

### 概要

ルーレット結果表示後、特定のプレイヤー1人分だけジョブを再抽選できる機能。
シンプルモード・高機能モードの両方で利用可能。

### UI仕様

| 要素 | 説明 |
|------|------|
| 再抽選ボタン | 各結果カードの右端に🔄アイコンのボタンを表示 |
| ボタン位置 | ロール表示の右隣 |
| アニメーション | 再抽選時は対象のカードのみアニメーション（他のカードは静止） |

### 動作仕様

| 項目 | 動作 |
|------|------|
| ロール維持 | 再抽選してもロール（タンク/ヒーラー/DPS）は変わらない |
| サブロール維持 | 高機能モードで設定されたサブロール（メレー/ピュア等）も維持 |
| 除外ジョブ | 高機能モードの除外設定が再抽選時も適用される |
| 重複禁止 | 高機能モードの重複禁止がONの場合、他メンバーと同じジョブは選ばれない |

### 制約事項

- 再抽選可能なジョブがない場合はエラーメッセージを表示
  - 例：重複禁止ONで全ての同ロールジョブが使用済みの場合
- 「ルーレット開始」で全体を再実行した場合は、再抽選の履歴はリセットされる

## 技術仕様

| 項目 | 内容 |
|------|------|
| フレームワーク | Vite + TypeScript |
| UI | バニラTS（フレームワークなし） |
| ホスティング | GitHub Pages |
| 状態管理 | クラスベースのローカル状態 |
| データ永続化 | なし（リロードでリセット） |
| テスト | Vitest |

### ファイル構成

```
src/
├── main.ts                         # メインアプリケーション（状態管理・イベント処理）
├── roulette.ts                     # ルーレットロジック（シンプル・高機能両対応）
├── roulette.test.ts                # ルーレットロジックのテスト
├── types.ts                        # 高機能モード用の型定義
├── data/
│   └── jobs.ts                     # ジョブデータ定義・ロール型定義
├── components/
│   ├── PlayerInputsRenderer.ts     # プレイヤー入力欄のレンダリング
│   ├── ResultsRenderer.ts          # 結果表示のレンダリング
│   └── modals/
│       ├── ExcludeJobModal.ts      # 除外ジョブモーダル
│       └── RoleModal.ts            # ロール選択モーダル
└── styles/
    ├── index.css                   # CSSエントリポイント（@import集約）
    ├── variables.css               # CSS変数（カラー・共通値）
    ├── base.css                    # リセット・レイアウト
    ├── buttons.css                 # ボタン系スタイル
    ├── forms.css                   # 入力フォーム・設定系
    ├── results.css                 # 結果表示・エラー
    ├── modals.css                  # モーダル共通・ジョブ/ロール選択
    └── responsive.css              # レスポンシブ対応（480px以下）
```

### モジュール責務

| モジュール | 責務 |
|-----------|------|
| `main.ts` | アプリ全体の調整役・状態管理・イベントハンドリング |
| `roulette.ts` | ルーレットロジック・パーティ構成管理 |
| `roulette.test.ts` | ルーレットロジックのユニットテスト |
| `types.ts` | 高機能モード用の型定義・デフォルト設定生成 |
| `data/jobs.ts` | ジョブデータ・ロール型・ラベル定義 |
| `components/PlayerInputsRenderer.ts` | プレイヤー入力欄のHTML生成 |
| `components/ResultsRenderer.ts` | 結果表示のHTML生成・エラー表示 |
| `components/modals/ExcludeJobModal.ts` | 除外ジョブモーダルの状態・表示管理 |
| `components/modals/RoleModal.ts` | ロール選択モーダルの状態・表示管理 |

## テスト

### テストフレームワーク

Vitest を使用。

### テスト実行

```bash
npm test             # 一回実行
npm run test:watch   # ウォッチモード
```

### テストケース一覧（roulette.test.ts）

| カテゴリ | ケース | 検証内容 |
|----------|--------|----------|
| 標準構成維持 | 汎用枠混在 | ヒーラー→バリア、DPS→キャス、フリー→タンクに解決 |
| 標準構成維持 | 汎用ヒーラーx2 | 1ピュア+1バリアに解決 |
| 標準構成維持 | 汎用DPSx4 | 2メレー+1レンジ+1キャスに解決 |
| 標準構成維持 | フリーx8 | 標準構成に解決 |
| 標準構成維持 | サブロール全指定 | そのまま維持 |
| 標準構成維持 | 汎用ヒーラーx1（バリア不足） | バリアに解決 |
| 標準構成維持 | 汎用DPSx1（キャス不足） | キャスに解決 |
| 標準構成維持 | 汎用DPSx2（レンジ+キャス不足） | レンジ+キャスに解決 |
| canMaintainStandardComposition | 超過 | falseを返す |
| canMaintainStandardComposition | 正常 | trueを返す |
| ジョブ重複禁止 | 重複禁止ON | 全員異なるジョブ |
| rerollSingleMember | 基本動作 | 1人分だけ再抽選できる |
| rerollSingleMember | 重複禁止ON | 他メンバーと重複しないジョブが選ばれる |
| rerollSingleMember | 除外ジョブ | 除外設定が考慮される |
| rerollSingleMember | 無効なインデックス | エラーを返す |
